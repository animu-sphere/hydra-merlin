# v0.1.0 - Reproducible renderer foundation

**Release date:** 2026-07-12
**Tag:** `v0.1.0`

## Objective

v0.1.0 establishes hdMerlin's host-neutral renderer foundation, reproducible
Core SDK packaging, and the release automation required for stable SemVer tags.

## Completed capabilities

- Host-neutral `RenderWorld` scene model and deterministic draw extraction with
  no OpenUSD, Hydra, DCC, Qt, or Vulkan types in the Core public API.
- Persistent Vulkan 1.4 offscreen renderer with color/depth CPU readback,
  multi-frame contexts, revision-based uploads, and shared completion for render
  products.
- Opt-in Hydra 2 adapter for mesh topology, transform, visibility, camera sync,
  USD path mapping, CPU RenderBuffers, and a Vulkan-backed render pass.
- Deterministic `merlin-benchmark` JSON reports with environment metadata,
  per-stage CPU timings, structural counters, and first-frame, steady-state, and
  scene-edit baselines.
- Versioned CMake package exports for `Merlin::RenderWorld`,
  `Merlin::RenderExtraction`, and optional `Merlin::Vulkan` consumers.
- Runtime-only packaging for headless, benchmark, and Hydra products so OpenUSD
  remains out of the exported Core/Vulkan dependency graph.
- Installed `merlin-release-metadata.json` recording schema and project versions,
  dependency constraints, configured feature layers, exported targets, and
  runtime-only products.
- Stable SemVer tag-driven Windows/Linux Core SDK release automation with
  project-version validation, release metadata verification, and SHA-256 checksum
  assets.

## Compatibility

- Core-only builds require CMake 3.24 and a C++20 compiler.
- Windows x64 with Visual Studio 2022 is the validated development path.
- Hosted Windows and Linux CI exercise Core-only Debug and Release builds plus
  isolated install-tree consumer tests.
- Vulkan/headless source builds require a Vulkan 1.4-capable graphics queue and
  `glslc` from a compatible Vulkan SDK.
- Hydra 2 builds require the Vulkan layer and a compatible OpenUSD 26.05 SDK.

## Known limitations

- Authored Hydra materials, MaterialX translation, native Hydra instancing, and
  subdivision refinement are not available; meshes use the fallback material.
- Advanced viewport features such as alpha mask/blend, dome lighting, shadows,
  selection, transparency, and production culling are not available.
- Vulkan/Hgi external-memory or zero-copy host presentation is not available;
  Hydra uses CPU RenderBuffer readback followed by host Hgi upload.
- OpenUSD shared/static mode, Debug/Release configuration, MSVC runtime, and
  plugin ABI compatibility are operator responsibilities in this release.
- A repository-scoped Windows GPU runner has not yet been enrolled, so
  Vulkan/Hydra capability workflow evidence is manual rather than continuous.

## Verification evidence

- Hosted Core CI for the release commit covers Windows and Linux Debug/Release
  Core builds, source-tree tests, and isolated install-tree consumers.
- The manually dispatched capability workflow contract covers Windows x64
  Vulkan/headless Debug and Release and Hydra Release on a runner labeled
  `vulkan-1.4`.
- Release tag verification rejects non-`vMAJOR.MINOR.PATCH` refs and tags whose
  version differs from `project(hdMerlin VERSION ...)`.
- Release metadata verification checks schema/version identity, feature flags,
  exported targets, runtime product names, and the runtime-only packaging
  contract.
- Granular delivery evidence remains archived in
  [delivery history](../reports/delivery-history.md), and the release pipeline
  hardening review is captured in
  [2026-07-12 release-foundation review](../reports/releases/01-2026-07-12-v0.1.0-release-foundation-review.md).
