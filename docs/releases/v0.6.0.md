# v0.6.0 - Incremental Hydra synchronization

**Release date:** 2026-07-15
**Tag:** `v0.6.0`

## Objective

v0.6.0 makes Hydra scene updates observable and resource-granular. The release
separates host, renderer, Vulkan, readback, and presentation costs, then keeps
Hydra source state and derived renderer resources stable enough for localized
mesh edits to avoid unrelated work.

## Completed capabilities

- `merlin-benchmark/v3` CPU/GPU stage distributions, hitch summaries,
  structural comparison reports, fixed scale/AOV/4K fixtures, and optional
  controlled-hardware timing thresholds.
- Versioned Hydra performance JSON and raw OpenUSD Chrome traces covering
  delegate Sync, scene-index processing, RenderWorld/extraction, Vulkan
  execution, readback, CPU-to-Hgi upload, composite, and presentation scopes.
- Persistent per-USD-path Hydra topology, points, primvar descriptors and
  values, normalized payloads, triangulation, material, transform, and
  visibility state across Sync calls.
- Locator-aware points, topology, primvar, transform, visibility, camera, and
  material-parameter fast paths with conservative fallback for coarse or
  unavailable dirty information.
- Per-aspect source and derived resource revisions plus changed vertex/index
  ranges preserved through RenderWorld, extraction, and Vulkan residency.
- Exact partial vertex/index uploads when resident base revisions are compatible,
  with safe full-upload fallback when topology, shape, or resident revisions do
  not match.
- Host-neutral `merlin-diagnostic/v1` records with stable codes, source paths,
  dispositions, and named recovery actions, bridged to Hydra diagnostics and
  telemetry.
- A documented OpenUSD 26.05 Gaussian ingestion boundary using the standard
  `ParticleField3DGaussianSplat` and Hydra `particleField` representation,
  without adding a custom USD schema or direct PLY/SPLAT parser.

## Compatibility

- The v0.5.0 MaterialIR, Vulkan image/sampler residency, and basic Hydra
  material/light translation contracts are preserved.
- The v0.3.0 indexed and face-varying Hydra mesh primvar normalization contract
  remains the input boundary for mesh data.
- Core public APIs remain independent of OpenUSD, Hydra, DCC, Qt, and Vulkan
  types; raw Hydra state and OpenUSD diagnostics stay in adapter layers.
- Hydra configuration now requires the validated OpenUSD 26.05 shared SDK and
  rejects MSVC Debug builds when only Release OpenUSD libraries are available.
- Tier 0 CPU RenderBuffer readback remains the correctness, headless, CI, and
  host fallback path.

## Known limitations

- Persistent bindless resource tables, stable GPU Scene handles, dirty queues,
  and completion-safe delayed retirement remain v0.7.0 work.
- General MaterialX graph translation, shader generation, disk caching, alpha
  blending, dome and multi-light lighting, shadows, selection, production
  culling, subdivision refinement, and Gaussian rendering remain unavailable.
- Vulkan/Hgi external-memory and zero-copy presentation are not implemented;
  usdview still uses CPU readback followed by host Hgi upload.
- GPU-driven indexed submission, an opaque Visibility Buffer path, meshlet
  rendering, and Mesh Shader backends remain planned optimization work with the
  conventional Forward path as the reference and fallback.
- A repository-scoped Windows GPU runner has not yet been enrolled, so
  GPU/Hydra evidence remains local validation and manual capability workflow
  evidence rather than continuous hosted evidence.
