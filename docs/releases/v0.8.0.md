# v0.8.0 - Slang shader foundation and Vulkan parity

**Release date:** 2026-07-17  
**Tag:** `v0.8.0`

## Objective

v0.8.0 moves the Vulkan Forward shader source of truth from GLSL to Slang while
preserving the working renderer's output, fallback behavior, packaging, and
performance evidence. It establishes a stable shader-facing ABI and a Metal
compile gate before new shader families or a native Metal renderer expand.

## Completed capabilities

- Shared Slang modules now provide the conventional and bindless Vulkan Forward
  vertex and fragment paths; the superseded GLSL runtime source is removed.
- The existing descriptor-indexing capability negotiation continues to choose
  bindless Forward when supported and conventional Forward otherwise.
- A backend-neutral shader capability and permutation contract keeps Core free
  of Vulkan, Metal, command, and resource-object concepts.
- Reflected C++/Slang checks validate Forward push-constant and material-uniform
  layouts plus entry points, resource classes, descriptor sets, and bindings.
  Mismatches produce actionable diagnostics.
- Slang 2026.8.x produces dependency-aware SPIR-V and Metal compile-gate
  artifacts. Versioned `shaders/v1` build-tree and install-tree packages include
  target reflection and deterministic SHA-256 manifests with compiler, profile,
  capability, matrix-layout, optimization, entry-point, permutation, generator,
  and dependency provenance.
- Metal declares non-uniform bindless resource indexing unsupported and uses the
  conventional Forward compile gate without weakening the Vulkan bindless path.
- Reference artifacts compare forced-conventional and automatic-path color,
  depth, `primId`, and `instanceId` output; IDs remain exact.

## Compatibility

- Core public APIs remain independent of OpenUSD, Hydra, Vulkan, Metal, Qt, and
  DCC SDK types.
- Vulkan 1.4 remains the minimum renderer contract. Conventional Forward remains
  the correctness and capability fallback.
- OpenUSD 26.05 remains the validated Hydra SDK/runtime boundary.
- This release does not add a Metal renderer or presentation path, MaterialX
  translation, Gaussian shaders, GPU-driven submission, Visibility, or meshlets.

## Verification evidence

- The repository-scoped Windows x64 `vulkan-1.4` runner passed Debug and Release
  Vulkan capability jobs and the Release OpenUSD 26.05 Hydra job in
  [capability run 29508228337](https://github.com/animu-sphere/hydra-merlin/actions/runs/29508228337).
- The capability workflow runs shader ABI and contract tests, Vulkan offscreen
  and validation tests, deterministic artifact checks, and install-consumer
  coverage. It retains validation logs, expected/actual/diff images, shader
  packages, and renderer reports.
- Release evidence built the `v0.7.0` tag on the same GPU. The structural
  benchmark comparison found zero regressions; the 120-frame steady-state GPU
  median changed from 1,231,700 ns to 1,188,085 ns ($-3.5\%$). A 20% timing
  threshold is retained as a non-gating observation.
- The Hydra job passed plugin discovery, CPU RenderBuffer, and install-tree
  usdview smoke coverage against OpenUSD 26.05.
