# v0.2.0 - Resource-granular GPU scene

**Release date:** 2026-07-13
**Tag:** `v0.2.0`

## Objective

v0.2.0 replaces whole-scene extraction and upload with immutable,
resource-granular snapshots and persistent suballocated Vulkan geometry. The
release makes edit cost depend on the resource that changed rather than the
size of the complete scene.

## Completed capabilities

- Immutable `FrameSnapshot` records split into geometry, material, instance,
  and draw resources, keyed by generation-safe serialized handles and
  per-resource revisions.
- Independently revisioned points and topology payloads shared across snapshots
  and instances.
- Device-local vertex and index arenas with first-fit suballocation, a
  persistently mapped staging ring, in-place size-preserving updates, and
  completion-based range retirement.
- Zero geometry upload for transform-, visibility-, and material-only edits;
  points-only edits stage only the changed packed vertex payload.
- `merlin-benchmark/v2` shared-geometry fixture and separate transform,
  visibility, material, points, and removal edit baselines.
- Geometry-cache, suballocation, release, retirement, and arena statistics for
  deterministic CI assertions.

## Compatibility

- This release is a breaking API change for extraction and Vulkan backend
  consumers: `ExtractedScene` was replaced by `FrameSnapshot`.
- The host-neutral Core boundary and optional Vulkan/Hydra dependency layering
  established in v0.1.0 are unchanged.
- Vulkan requirements remain a Vulkan 1.4 device and compatible `glslc`.

## Known limitations

- Rendering remains synchronous at the public backend boundary; request,
  submission, completion, and resolve are not yet separate operations.
- Render targets and CPU readback are fixed rather than selected per request.
- Authored material networks, MaterialX translation, native Hydra instancing,
  and practical indexed/face-varying primvar normalization remain future work.

## Verification evidence

- `merlin-vulkan-resource-update` verifies zero steady-state work, shared
  geometry, per-aspect edit costs, deterministic removal, and generation-safe
  handle reuse on a Vulkan-capable machine.
- Benchmark JSON validation checks the v2 baseline order, counters, and
  resource-specific edit invariants.
- Core-only Debug/Release and install-tree package consumers remain covered by
  the hosted CI contract.
