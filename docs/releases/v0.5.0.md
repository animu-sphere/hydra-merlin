# v0.5.0 - MaterialIR and basic shading

**Release date:** 2026-07-14
**Tag:** `v0.5.0`

## Objective

v0.5.0 establishes one host-neutral material boundary for headless and Hydra
inputs and renders a deliberate textured, directional-lit raster subset through
the persistent Vulkan backend.

## Completed capabilities

- Revisioned MaterialIR parameter blocks, textures, samplers, feature masks,
  alpha/cutoff and double-sided state in the host-neutral scene model.
- Deterministic extraction records and structured material fallbacks for
  missing, stale, or unsupported bindings.
- Completion-safe Vulkan image/sampler residency and feature-keyed shader,
  descriptor-layout, and pipeline caches.
- Base and vertex color, display opacity, normals, UV RGBA8 textures,
  directional lighting, opaque shading, and alpha masking.
- Per-frame material uniforms so value-only edits reuse the existing shader and
  pipeline variant.
- A basic Hydra `UsdPreviewSurface`/`UsdUVTexture` and distant-light translation
  using the same MaterialIR path as headless rendering.
- Textured headless expected/actual/diff evidence and an install-tree textured
  usdview regression.
- OST 0.17.0 capability-CI pinning and current managed/external
  `ost renderer view` operating guidance.

## Compatibility

- The v0.4.0 explicit request/submit/completion/resolve contract is preserved.
- The v0.2.0 resource-granular FrameSnapshot and v0.3.0 Hydra mesh contracts are
  preserved.
- Core public APIs remain independent of OpenUSD, Hydra, DCC, Qt, and Vulkan
  types; raw Hydra and future MaterialX graphs remain outside Core.
- Tier 0 CPU RenderBuffer readback remains the correctness, headless, CI, and
  host fallback path.

## Known limitations

- General MaterialX graph translation, shader generation, and disk caching are
  not implemented.
- Alpha blending, dome and multi-light lighting, shadows, selection, production
  culling, subdivision refinement, and Gaussian rendering remain unavailable.
- Vulkan/Hgi external-memory and zero-copy presentation are not implemented;
  usdview still uses CPU readback followed by host Hgi upload.
- A host-neutral diagnostic sink and strict OpenUSD build/runtime ABI rejection
  remain cross-cutting roadmap work.
- GPU/Hydra validation is locally exercised but the manual capability workflow
  does not yet have retained GitHub-run evidence from a continuously available
  self-hosted runner.
