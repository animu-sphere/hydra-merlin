# v0.3.0 - Practical Hydra mesh input

**Release date:** 2026-07-13
**Tag:** `v0.3.0`

## Objective

v0.3.0 turns the Hydra adapter from a foundation smoke path into a practical
mesh ingestion path, including robust polygon normalization, native instancing,
and object-ID render products suitable for host selection workflows.

## Completed capabilities

- Normal, display color/opacity, and UV normalization for constant, uniform,
  vertex, varying, and face-varying Hydra primvars, including indexed data.
- Deterministic ear-clipping triangulation for concave polygons with actionable
  diagnostics for holes, malformed indices, degeneracy, and self-intersection.
- Native and nested Hydra PointInstancer transforms, including translate,
  rotate, scale, and matrix primvars, flattened to shared Merlin geometry.
- `primId` and `instanceId` Vulkan attachments, tightly packed CPU readback,
  and Hydra `HdFormatInt32` RenderBuffer delivery.
- Authored material-binding identity retention with correct adapter-side
  material lifetime tracking; shading still uses the fallback material path.
- Scale gates for one million triangles, 10,000 small meshes, and 256 repeated
  primvar-only edits.
- Root `VERSION` source of truth and simplified `prepare-release` bookkeeping.

## Compatibility

- The v0.2.0 resource-granular `FrameSnapshot` and GPU residency contracts are
  preserved.
- Hydra builds continue to require a compatible OpenUSD 26.05 SDK and the
  optional Vulkan layer; Core/Vulkan package consumers do not acquire OpenUSD
  as a transitive dependency.
- Release identity is now synchronized through `VERSION`, CMake project
  metadata, installed metadata, and changelog comparison links.

## Known limitations

- Material-network shading and MaterialX translation are not implemented.
- Subdivision refinement, alpha modes, lighting, shadows, selection policy,
  and production culling remain unavailable.
- Hydra presentation remains CPU RenderBuffer readback followed by host Hgi
  upload; there is no Vulkan external-memory or zero-copy path.
- Backend execution is still exposed as one synchronous `Render` operation.

## Verification evidence

- Adapter regression coverage includes indexed face-varying primvars, concave
  polygons, material binding lifetime, native/nested instancing, and the scale
  fixtures listed above.
- `merlin-render-product-contract` checks canonical color, depth, primId, and
  instanceId metadata, row pitch, payload size, finite normalized depth, and a
  nonzero completion value.
- Install-tree Hydra discovery, RenderBuffer lifetime, usdview first-frame, and
  stable-update tests remain the host validation contract.
