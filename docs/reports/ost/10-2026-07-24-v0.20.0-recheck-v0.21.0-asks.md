This pass rechecks the v0.20.0 renderer-workflow asks and exercises both
`ost renderer view` and `ost renderer viewport` with the Kitchen Set USD scene.
The repository HEAD still contains the v0.19.0 project integration, while the
installed OST CLI is v0.20.0.

## Summary

The v0.20.0 CLI resolves the declared `viewport-usd` intent and its scene
capability preflight. `ost build --check` also passes, and redacted dry-run
output remains machine-readable. A real managed renderer launch is still
blocked: the first `renderer view` attempt failed because the generated
`CMakeUserPresets.json` included a missing core preset, and after generating
the core preset both managed renderer workflows stalled during the MSVC ABI
try-compile. No usdview or standalone viewport window launched, no durable
viewport launch record was produced, and no successful renderer JSON envelope
was captured.

## Reproducible commands

```powershell
ost --version
ost build --check --json
ost build --dry-run --intent viewport-usd --profile usd --json --redact-paths
ost renderer viewport --intent viewport-usd --profile usd --preflight --json --redact-paths -- `
  --usd <scene-root>/Kitchen_set/Kitchen_set/Kitchen_set.usd --frames 1 --hidden
ost renderer view --profile usd --json --redact-paths `
  <scene-root>/Kitchen_set/Kitchen_set/Kitchen_set.usd
ost renderer viewport --intent viewport-usd --profile usd --json --redact-paths -- `
  --usd <scene-root>/Kitchen_set/Kitchen_set/Kitchen_set.usd --frames 1 --hidden
ost validate --json
```

The scene-aware preflight returned `workflow: usd-scene`, selected the
`cy2026-windows-x86_64-py313-usd` target, and applied `usd-stage-read`. The
two launch commands did not produce a window or a successful JSON launch
envelope because managed CMake configuration did not complete.

## v0.21.0 asks

1. Make managed configure diagnostics include the active child, phase, and
   retained CMake log tail, and ensure timeout cleanup removes descendants and
   target leases deterministically.
2. Make generated preset includes self-contained when a profile is selected;
   `renderer view` must not fail because an unrelated core preset is absent.
3. Complete and persist the renderer JSON envelope for both `view` and
   `viewport`, including build, executable, readiness, backend/device, child
   exit, and redacted launch-session evidence.
4. Bind managed renderer reports and completion records to an OST-owned
   producer session so a stale external report cannot be promoted to PASS.

Raw JSON, logs, runtime paths, and machine-specific process details remain
local and are intentionally not attached to this report.
